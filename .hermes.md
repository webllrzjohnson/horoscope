This is Louie's horoscope project.

## Project Defaults

- Path: `D:\Factory\horoscope`.
- Treat this as a regular Factory workspace project.
- Inspect the current stack and implementation before assuming framework, database, auth, deployment, or content-generation patterns.

## Working Rules

- Inspect existing files, package scripts, documentation, environment examples, and project structure before editing.
- Make the smallest safe change that solves the task.
- Follow existing UI, content, and code style.
- Do not add new external services, payment providers, databases, analytics, or AI APIs unless Louie approves.

## Content / Product Rules

- Keep horoscope content user-friendly, clear, and non-alarming.
- Avoid presenting horoscope/astrology output as medical, legal, financial, employment, or safety advice.
- If generating public content, keep tone polished, concise, and suitable for a consumer-facing site.
- Preserve any existing house style or prompt format found in the repo.

- `npm run build`
